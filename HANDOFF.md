## 2. 英语项目：魔法英语乐园

### 2.1 文件结构

```
english-game/
├── index.html      骨架 + 全部 CSS（单文件，无框架）
├── data.js         唯一内容源（UNITS/PHONICS/PETS/CARE/OUTFITS/STICKERS/THEMES 等）
├── app.js          全部逻辑（~3000 行，纯 JS，无依赖）
├── audio/          422 条真人发音 mp3 + manifest.js（文本→文件名索引）

```

**技术栈：纯前端，零依赖，零构建。** 改完直接 `git push`，GitHub Pages 自动部署。

### 2.2 内容规模

| 册别 | 单元 | 单词 |
|---|---|---|
| 二年级（基础词） | 2 | 31 |
| 三上 | 6 | 55 |
| 三下 | 6 | 48 |
| 四上 | 6 | 71 |
| 四下 | 6 | 68 |
| **合计** | **26** | **273** |

- 句型 102 句 · 自然拼读 8 条规则 / 64 个例词（带音标 + 音节拆分）
- **注意：人教 PEP 是三年级起点，官方没有二年级英语教材。**「二年级」那两个单元收的是各版本共有的基础词（数字/颜色/身体/家人）。

### 2.3 核心机制（改动前必须理解）

**① SRS 间隔重复引擎（最值钱的部分）**
```js
S.srs[word] = { lv: 1~6, due: "YYYY-MM-DD" }
SRS_STEPS = [1, 2, 4, 7, 15, 30]   // 天
```
- 到期复习**答对** → 等级 +1，间隔拉长；**答错** → 打回 1 级，明天重练
- **防刷关键**：`srsGrade()` 里，**未到期的词答对不升级**（`if (s.due > todayStr()) return;`）
  否则孩子一天刷十遍同一个词就能冲到满级，整个调度器就成了摆设。**不要删这行。**

**② 今日复习的选词规则（家长明确指定的）**
```js
dueWords() = 昨天刚学的（全部） + SRS到期的 + 不足6个时从「今天以前」学过的老词里随机补
```
- ⚠️ **`oldRandom()` 必须排除「今天学的词」**（`S.learnedAt[w] < todayStr()`）
  否则会让她复习 2 分钟前刚学的词——那不是复习，是重复。这是修过的 bug。

**③ 每日任务（顺序不能乱）**
```
① 今日复习 → ② 学新词 → ③ 玩游戏 → ④ 自然拼读（每天 1 条）
```
- 首页每一行**可点击直接跳转**到对应模块
- **四项全部完成，才能转当天的转盘**（转盘一天只能转一次，`S.daily.spun`）
- 拼读**只计 `ph`，不计 `g`**——否则做 3 次拼读就凑满了游戏任务，单词游戏可以完全不碰

**④ 难度三段自动升（`DIFFS` / `levelNum()` / `D()` / `distract()`）**
| 段位 | 触发 | 关键差异 |
|---|---|---|
| 🌱 小小魔法学徒 | 0~24 | 3新词 / **3选1** / **不倒计时** / 拼写≤6字母+"偷看3秒" |
| ✨ 魔法师 | 25~59 | 4新词 / 4选1 / 12秒 |
| 🦄 大魔法师 | 60+ | 5新词 / **8秒闪电轮** / 拼写全开 |

- **所有游戏的选项数、题量、时限都必须走 `D()`**，不要写死
- **`masteredCount()` 里低年级词只算半个权重**（`CORE_BOOKS = ["四上","四下"]`）
  否则暑假刷完二年级+三上（86个简单词）就会把难度顶到最高，可她一个四年级新词都没学。这是修过的 bug。
- **「对新手，倒计时不是刺激，是恐吓」**——8秒闪电轮曾经是所有人一进来就有的，那是我最大的失误。现在只在 3 段出现。

**⑤ 游戏厅题库 = `gamePool()`，只出「学过的词」**
- ⚠️ 不要用 `unlockedWords()` 做游戏题库——每册第一单元都默认解锁，会考她**从没学过**的词，纯粹是挫败。这是修过的 bug。

**⑥ 伙伴系统（猫小九/迈克狐/猴子警长）**
- 三个角色 + 独角兽，各有独立进化线；**孩子可以自己改名字**
- 喂养：饱腹/干净/心情三条状态 + 亲密度
- **红线：绝不惩罚。** 状态只会变淡（地板 20），**不会生病、不会死、不扣任何东西**
  养宠物是为了让她想回来，不是为了制造愧疚和焦虑。**不要加"宠物生病了""快饿死了"这类机制。**
- **伙伴形象可由家长上传图片**（存 localStorage，**不进仓库、不上传服务器、不进备份码**）
  饰品用「拖动 + 放大缩小 + 旋转」编辑器贴合任意图片（`S.pet.deco[petId][slot] = {x,y,s,r}`）

**⑦ 跨科目共享钱包**（见第 4 节）

### 2.4 发音（踩过大坑，已定稿）

- ⚠️ **不要用手机自带 TTS**。家长实测：很多手机没装英文语音包，孩子只能听到"叮"的音效，听不到单词。
- 现在：**edge-tts 预合成 422 条 mp3**，`speak()` 先播 mp3，系统 TTS 只作兜底。
- **声音已定稿：`en-US-JennyNeural`（单词 -15%，句子 -10%）。家长明确表示满意，不要再换。**
- 🔴 **改了 data.js 加了新单词/句子，必须重跑发音合成脚本，否则新词是哑巴：**
  ```bash
  python3 -m venv .venv && .venv/bin/pip install edge-tts
  .venv/bin/python tools/gen_audio.py          # 换声音：加参数 tools/gen_audio.py en-US-AriaNeural
  ```
- iOS 坑：音频必须在**真实手势**里先播一次静音才解锁；`cancel()` 后要**延迟一帧**再 speak。
- 离开页面必须 `stopSpeak()`（pause mp3 + cancel TTS + 递增 `speakGen`），否则退出游戏后还在念。
  `speakThen()` 的回调**必须校验 `speakGen`**，否则离开后音频 `onended` 迟到触发，会在后台偷偷推进题目。

### 2.5 平台限制（跟读功能踩过两次坑）

- **iOS 的 Safari/Chrome 都不支持 Web 语音识别**（只支持合成）
- **★ iPhone「添加到主屏幕」后从桌面图标打开时，iOS 不给麦克风权限** ← 用户实测确认
  → **玩跟读必须用 Safari 直接打开网址**，其他功能用桌面图标没问题
- 微信内置浏览器：能玩、发音正常，但不能加桌面、跟读用不了
- 「魔法回声」跟读做了三模式自适应：`sr`（安卓Chrome自动打分）/ `record`（iPhone录音对比）/ `shadow`（无麦克风的影子跟读）
  **麦克风任何失败都自动降级到 shadow，绝不卡死**
- 家长后台有「🎙️ 跟读自检」页，远程排查先看这页

## 3. 语文项目：寻宝作文记

### 3.1 文件结构

```
chinese-game/
├── index.html
├── data.js         唯一内容源（TOOLS/STOPS/IDEAS/ESSAYS/BUDDY）
├── check.js        ★ 技巧检测引擎（项目心脏）
├── app.js
├── tests/          flow.js(63) / judge.js(29) / admin.js(61) / wallet.js(13)
├── sw.js / manifest.json
```

### 3.2 核心设计（这个项目的命门）

**把语文装进她已经爱着的壳子里** —— 她爱看《寻宝记》，所以整个产品做成一场寻宝：
12 座城市（桂林/北京/西安/厦门/成都/敦煌/上海/杭州/洛阳/拉萨/哈尔滨/三亚）
每站：**4 张知识卡**（她本来就爱看）→ **5 个微写作任务** → 收进「宝库」

**她不缺内容——她脑子里全是寻宝记的知识，只是没人把它和"写作文"接上。**

**① 五大题材（同事的建议，最有价值的一条）**
每座城市的 5 个任务分别是 **景 / 物 / 人 / 事 / 食**——因为**小学作文考试就考这四大题材**（写人/写景/状物/写事）。
> 技巧是横轴（六件法宝），题材是纵轴。**两个都要练。**

**② 六件法宝（写作技巧）**
比喻杖 / 五感镜 / 分解刀（动作分解）/ 读心术（心理描写）/ 环境笔 / 对话铃

**③ ★ 技巧检测引擎 `check.js` —— 整个项目的心脏**

```
只回答「用没用这个技巧」（客观可判），绝不回答「写得好不好」（判不准，硬判会伤害一个本来就抗拒作文的孩子）
```

- **防错判是生命线**（中文没有分词，子串误伤）：
  - `EXCLUDE` 上下文排除表：「**辣**椒」里的辣不算味觉、「**热**闹」里的热不算触觉……
  - 比喻词分强弱：「一样」**只有和 像/跟/和/如/似 搭配时**才算比喻（否则「捡到**一样**东西」会被误判）
  - 动词表已剔除高危单字（张/背）
- **反馈原则**：
  - 没用上技巧**也给 1 星**（写了就有分）
  - 标题是「**写出来了！还差一点点**」，**绝不否定她**
  - 就算没用上技巧，**也允许直接收进宝库拿金币**（绝不强迫改）
- ⚠️ **改这个文件前必须先跑 `node tests/judge.js`（29 项，含防错判用例）**

**④ 脑洞任务（破抗拒的核武器）**
「给李白发一条朋友圈，他会怎么回？」——**只看字数，只夸不批评**。
**先让她愿意写，再谈写得好。** 不要给脑洞题加任何检测和批评。

**⑤ 宝库（读→摘→记→用 闭环）**
她写的每一句都存进宝库 → **周末写作文时可以一键调用**。

**⑥ ★ 作文批阅台（语文家长后台的灵魂）**
系统能判「用没用比喻」，**判不了「写得好不好」——那只有人能给**。所以：
- 家长读全文 → 打分 → **写评语（不写不让提交，评语比分数重要）**
- 系统在旁边列出「检测到的技巧」供参考，但明确写着「**这只是用没用，不代表写得好不好。好不好，只有你能判。**」
- 教家长怎么批：**先说一句你真心喜欢的（这比指出十个毛病有用），再提一个可以改进的点，只提一个**
- **家长批阅完，孩子在作文页能看到评语，并再拿 2 张转盘券** → 所以她会主动催家长看

**⑦ 搭档「小獾」**
她拒绝单向灌输，所以必须有一个「人」在跟她说话，而不是冷冰冰的分数。每次提交都有小獾的回应。

### 3.3 关于「全国地图点击闯关」（家长同事的建议）

**我没做，并且建议后续也不要手绘中国地图。**
国界（台湾、南海诸岛等）画错是严重问题，一个学习网站上出现有误的中国地图，风险远大于收益。
**替代方案**：手绘风格的**探险路线图**（曲线路径 + 城市站点，不追求地理精确），真正的地理知识放在知识卡里讲。

## 4. ★ 跨科目共享钱包（改任一项目的金币逻辑前必读）

```js
localStorage["sharedWallet_v1"] = { coins: N, tickets: N }
```

两个 App 同域，**共用同一份金币和转盘券，喂同一只宠物**。

- **英语侧**：`walletIn()` / `walletOut()`；`S.walletMigrated` 一次性并入旧余额（**防止重复计数**）；`save()` 时写钱包；从后台回前台时重读并提示「语文那边赚的金币到账啦」
- **语文侧**：`loadWallet()` / `saveWallet()`；`addCoins`/`addTicket` 都**先重读钱包再写**（英语可能刚改过）
- **效果**：语文写完作文发的转盘券，能在英语App的幸运大转盘上直接转
- ⚠️ **改任一 App 的金币逻辑，必须跑 `chinese-game/tests/wallet.js`（13 项：验证互通 / 不重复计数 / 不互相覆盖）**

- ⚠️ **但先别急着做**：如果她连脑洞题都不肯写，阅读模块做了也白做。**先看她愿不愿意动笔。**

**② 「密室逃脱」式解谜关**（家长同事的建议）
- 每座城市进城前有一道解谜题，**答案藏在知识卡里**——逼她真的把知识卡看进去
- 三亚站已经做成了「海岛逃生」主题（椰子救命 / 挖沙坑取淡水 / SOS 求救），可以从这站开始试

**③ 探险路线图**（替代"中国地图"）
- 手绘风格的曲线路径 + 城市站点 + 宝箱图标，不追求地理精确（**不要画中国国界**）

### 7.3 其他已知可优化点

- **英语**：拼读只有 8 条规则，已加「拼读大挑战」（混合复习）兜底，但规则本身可以扩充（如 `ar/or/ir/er/ur`、`ch/sh/th`、`oa/ow`）。**扩充后记得重跑发音合成。**
- **英语**：句型只有 102 句，四上/四下比较全，低年级偏少。
- **语文**：目前只有 6 个部编版习作主题（四上/四下各 3 个），可以补齐到 8 个。
- **两个项目**：家长手册（`~/Downloads/魔法英语乐园-家长手册.pdf`）已经落后于当前版本（单元页结构、家长密码、跟读限制等），需要重新生成。
  - 手册源文件在 scratchpad（临时目录，可能已丢失），**建议重写**
  - ⚠️ 生成中文 PDF 的坑：**无头 Chrome 打印 PDF 不嵌入 macOS 系统字体，文字会全空白**。必须用 `@font-face` 指向 `file:///System/Library/Fonts/Supplemental/Arial%20Unicode.ttf`，并用 `--headless=old --allow-file-access-from-files --print-to-pdf`。

## 8. 🔴 红线（不要碰）

1. **不要让系统去判断作文"写得好不好"**（语文）。判不准，硬判会伤害一个本来就抗拒作文的孩子。系统只判「用没用某个技巧」。
2. **不要给宠物加惩罚机制**（英语）。不生病、不死亡、不扣分。养宠物是为了让她想回来，不是制造愧疚。
3. **不要给新手加倒计时**（英语）。对还在认字母的孩子，倒计时不是刺激，是恐吓。
4. **不要删 SRS 的防刷判断**（英语 `srsGrade()` 里 `if (s.due > todayStr()) return;`）。
5. **不要把家长上传的伙伴图片提交进仓库**（那是别人的版权美术图，仓库是公开的）。`.gitignore` 里有双保险。
6. **不要手绘中国地图**（语文）。国界画错是严重问题。
7. **改了 data.js 加新词，必须重跑发音合成**（英语），否则新词是哑巴。
8. **每次发版 bump sw.js 的 CACHE 版本号**，否则用户看到的是旧版。

## 9. 工作方式建议

- 这个家长会**真机实测并给出反馈**（没声音、iPhone跟读死胡同、句子没读完就翻页、学词枯燥、饰品点不到……）
  **他的实测反馈比任何设计想象都值钱。认真对待每一条。**
- 改 UI 布局**别只靠想象**——用无头 Chrome 截图实测：
  ```bash
  "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=old --disable-gpu \
    --screenshot=out.png --window-size=500,900 "file:///path/to/harness.html"
  ```
  （注意：它可能不按给定宽度渲染，要在页面里量 `innerWidth` 核对）
- **加新功能后跑一下 `english-game/tests/audit.js`**——它是专门用来暴露逻辑漏洞的（游戏题库是否会考没学过的词、段位阈值是否失衡等）。

## 10. 最后

这两个网站解决的是**兴趣和记忆保持**，不是替代教材和课堂。

它们的价值在于：**让一个不喜欢英语和作文的孩子，愿意每天碰 15 分钟，并且把学过的东西真正留在脑子里。**

所以每次改动前，问一句：**这个改动会让她更想打开这个网站，还是更不想？**

祝顺利。
